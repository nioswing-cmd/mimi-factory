# 미미팩토리 3앱 개별 마케팅 플랜

> 2026-07-09 작성. 전제: 각 앱은 웹앱(무료 진열대)으로 트래픽을 모은 뒤, TWA로 구글플레이에 유료/구독 앱으로 등록한다.

## 전체 전략 한 장 요약

```
[콘텐츠 공장]  매일 새벽 자동 생산 (이미 가동 중)
     ↓
[무료 웹앱 3종]  트래픽·팬 확보 = 마케팅 자산
     ↓                    ↺ 공유 루프 (결과 카드, 카톡, 블로그)
[구글플레이 앱]  콘텐츠 20개+ 쌓이면 TWA 등록 → 유료화
```

핵심 원칙: **콘텐츠 자체가 광고다.** 매일 나오는 신작 하나하나가 검색 유입·공유 유입의 낚싯바늘. 마케팅 비용 0원으로 시작한다.

## 앱 1 — 퀴즈로읽는책 · 밤의 서재 (quiz.html)

- **타깃**: 독서·자기계발 관심층, 책은 사놓고 완독 못 하는 사람, 북클럽 멤버
- **최대 무기**: 🔑 **독서위키 블로그 (누적 200만 방문)** — 이미 가진 트래픽을 앱으로 돌리는 게 1순위
- **핵심 루프**: 블로그에서 책 리뷰/요약 글 → 글 끝에 "이 책, 퀴즈로 완독해보기 →" 앱 링크 → 앱에서 "다른 책 퀴즈 보기" → 서가 → 구독 배너로 블로그 회귀 (양방향 순환, 이미 링크 구축 완료)
- **실행**:
  1. 매일 신작이 나오면 블로그에 짧은 소개 포스팅 (제목: "『삼체』 3분 퀴즈로 완독하기" — 책 제목 검색 SEO가 목적)
  2. 티저(무료 10문항)를 공유 단위로: 북스타그램·독서 커뮤니티(카페, 오픈채팅)에 티저 링크 시딩
  3. 신간 출판사 협업: 자료 기반 생산 모드를 활용해 출판사 공식 자료로 퀴즈 제작 → 출판사가 마케팅 채널에 공유 (윈윈)
- **수익화**: 티저 무료 → 프리미엄(전체 문항+완독 리포트) 구독. 스토어 등록 후 월 구독.

## 앱 2 — 도파민 실험실 (test.html)

- **타깃**: MZ 테스트 놀이층 (MBTI·심리테스트 소비층), 인스타/틱톡 사용자
- **최대 무기**: 🔑 **결과 공유 카드** — 테스트 결과를 이미지 카드로 저장·공유하는 기능(스킬에 내장). 심리테스트는 "내 결과 자랑+친구 소환"이 유일하고 강력한 성장 엔진
- **핵심 루프**: 테스트 → 결과 카드 공유(인스타 스토리·카톡) → 친구가 링크 타고 유입 → 자기도 테스트 → 재공유
- **실행**:
  1. 신규 테스트 기획을 공유 욕구 중심으로: "나의 독서 유형", "도파민 중독 지수", "집중력 타입" 등 결과가 정체성이 되는 주제
  2. 시즌 테스트 캘린더: 신년(새해 다짐 유형) → 발렌타인(연애 유형) → 여름휴가(휴가 스타일) — 시즌 2주 전 생산
  3. 인스타 릴스: 결과 카드 4~6장 슬라이드 + "너는 어떤 타입?" — 제작 부담 최소 포맷
- **수익화**: 기본 무료 + 심층 리포트(상세 해설·처방) 유료. 스토어판은 광고 제거+전체 리포트 구독.

## 앱 3 — 둘의 피크닉 (friend.html)

- **타깃**: 커플, 친구 사이, 소개팅·첫만남, 가족 모임, MT/워크숍 진행자
- **최대 무기**: 🔑 **"둘이 같이 보는 한 화면"** — 상황(데이트, 술자리, 차 안, 명절)이 곧 검색어. 카톡으로 "이거 같이 하자" 보내는 순간이 전파 순간
- **핵심 루프**: 한 명이 발견 → 카톡으로 상대에게 링크 전송(OG 미리보기가 첫인상 — 이번에 설치 완료) → 둘이 플레이 → 각자 다른 상대와 재사용
- **실행**:
  1. 상황별 콘텐츠 라인업: "소개팅 첫 만남 카드", "차 안에서 하는 게임", "명절 가족 질문", "MT 아이스브레이킹" — 상황 검색 SEO
  2. 커플 커뮤니티·데이트코스 블로그 시딩, 데이트 앱 커뮤니티
  3. 오프라인 확장: 모임 진행자(교사, HR, 동아리)용 "진행용 대형 화면 모드" 소문내기
- **수익화**: 기본 무료 + 프리미엄 카드팩(심화 질문, 커플 전용). 스토어판 구독.

## 공통 인프라 (이미 완료 ✅ / 해야 할 것 ⬜)

- ✅ 각 앱 독립 랜딩 + OG 태그(공유 미리보기) + PWA 매니페스트
- ✅ UTM 체계: utm_source=quizbook/dopalab/picnic — 유입 채널 구분
- ✅ 매일 자동 생산 파이프라인 (콘텐츠 볼륨 = 시간이 벌어줌)
- ⬜ 앱 아이콘 PNG (512×512) 3종 — TWA 등록 필수, 디자인 필요
- ⬜ 방문 통계: GitHub Pages는 통계가 없으므로 무료 통계(예: GoatCounter, Cloudflare) 삽입 → UTM 효과 측정
- ⬜ 콘텐츠 20개+/앱 확보 (독서퀴즈는 근접, 테스트·친해지기는 시트에 기획 투입 필요)
- ⬜ 구글플레이 등록: 개발자 계정($25) → Bubblewrap으로 TWA 3개 패키징 → 심사
- ⬜ 유료 구독: 인앱결제(수수료 15~30%) vs 웹 구독(토스페이먼츠 등) 병행 검토
