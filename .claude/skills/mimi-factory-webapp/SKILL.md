# 미미팩토리 웹앱 (의미 × 재미 × 도파민)

미미팩토리의 모토는 **"의미와재미공장"**이다. 의미 있는 내용을 그냥 전달하지 않고, 사용자가 **손으로 만지고 → 즉시 반응을 받고 → 또 누르고 싶게** 만드는 것이 이 스킬의 전부다. 좋은 내용을 지루하게 만드는 건 실패이고, 재미만 있고 남는 게 없는 것도 실패다. 둘을 동시에 잡는다.

## 0. 가장 먼저 할 일 — 유형 판별

만들 결과물이 어떤 유형인지 먼저 정한다. 유형별 전용 스킬이 있으면 그것도 함께 읽는다. **단, 어떤 유형이든 아래 도파민 원칙과 브랜드 푸터는 이 스킬에서 가져온다.**

| 결과물 유형 | 함께 읽을 스킬 | 예시 |
|------------|--------------|------|
| 선택 → 결과 유형(진단·심리테스트·MBTI·궁합·매칭) | `dopamine-assessment-builder` | 성향 테스트, 궁합 |
| 책 1권 핵심 → 반복 퀴즈 + 완독 리포트 | `quiz-to-read-a-book` | 책 요약 퀴즈 |
| 발표·강의용 슬라이드 씬 | `presentation-scene-builder` | 영상 씬, 발표 자료 |
| 그 외 일반 인터랙티브 웹앱 | (이 스킬만) | 질문 카드, 계산기, 미니게임, 학습 도구, 추첨기 |

## 1. 의미를 재미로 바꾸는 4단계

지식·정보·메시지를 받으면 그대로 보여주지 말고 이렇게 변환한다.

1. **핵심 한 줄로 압축** — 이 결과물이 사용자에게 남겨야 할 단 하나의 메시지/효용을 먼저 정한다.
2. **행동으로 바꾸기** — 읽게 하지 말고 *누르게/뽑게/맞히게/움직이게* 한다. 정보 → 카드 뽑기, 설명 → 단계별 공개, 목록 → 슬롯/룰렛, 비교 → 토글 대결.
3. **즉각 보상 붙이기** — 모든 행동에 0.3초 안에 반응을 준다(아래 도파민 7요소).
4. **'한 번 더' 만들기** — 끝에 다시 하기/공유/다음 단계가 자연스럽게 이어지게 한다.

## 2. 도파민 7요소 (모든 웹앱 공통 체크리스트)

빌드 전·후로 이 표를 체크한다. 최소 4개 이상 적용한다.

| # | 요소 | 적용 방법 |
|---|------|----------|
| 1 | **즉각 반응** | 모든 탭/클릭에 팝 애니메이션 + 색·스케일 변화 (0.3~0.45s, `cubic-bezier(.34,1.56,.64,1)`) |
| 2 | **진행 시각화** | 상단 게이지/링이 실시간으로 차오르고, 찰 때 글로우(box-shadow)가 번진다 |
| 3 | **마일스톤 축하** | 단계 전환·완주 시 토스트 + 색 변화 + 진동으로 "해냈다" 순간을 만든다 |
| 4 | **변주(매번 다름)** | 누를 때마다 색/순서/문구를 살짝 바꿔 같은 화면이 반복되지 않게 한다 |
| 5 | **촉각 피드백** | 모바일 `navigator.vibrate()`로 뽑기 28ms, 축하 `[20,40,20]` 패턴 |
| 6 | **점층 구조** | 쉬운 것 → 깊은/어려운 것으로, 색이 따뜻→뜨겁게 깊어지게 (예: 노랑→코랄→핑크) |
| 7 | **완주 보상** | 마지막에 타이머·리포트·뱃지·공유 등 "끝까지 한 사람만 보는" 화면을 준다 |

> 진단/테스트형은 여기에 더해 `dopamine-assessment-builder`의 **"숨김+실시간 시각화"**(선택지 단서 숨기기 ↔ 상단 그래프 실시간 누적) 원칙을 반드시 함께 적용한다.

## 3. 디자인 토큰

자세한 토큰·코드는 `references/dopamine-patterns.md` 참고. 기본값:

- **타이포**: 디스플레이 `Jua`(둥근 한글) + 본문 `Noto Sans KR`. 구글폰트 CDN 사용. 강조 텍스트는 그라디언트 `background-clip:text`.
- **무드**: 다크 배경(딥 퍼플/네이비) 위에 따뜻한 포인트 그라디언트. 카드·글래스·글로우.
- **모션**: 팝(`scale + rotate`), 페이드 인, 게이지 글로우. `prefers-reduced-motion`은 항상 존중.
- **레이아웃**: 모바일 우선(`max-width:520px`, `min-height:100dvh`). 큰 액션 버튼은 하단 고정에 가깝게.
- **한 곳에 대담함**: 시그니처 요소(카드 뽑기/룰렛/링 등) 하나만 화려하게, 나머지는 조용하게.

## 4. 브랜드 푸터 (필수 · 항상 동일)

**모든 미미팩토리 웹앱 결과물 하단에 아래 푸터를 그대로 넣는다.** 코드 원본은 `assets/brand-footer.html`에 있으며, 복사해서 `</body>` 직전에 배치한다. 색·자간·그라디언트를 임의로 바꾸지 않는다.

구조: ① 상단 라벨 `의 미 와 재 미 공 장`(넓은 자간 + 양옆 페이드선) → ② 워드마크 `미미팩토리`(시안→블루→퍼플→핑크 그라디언트, Jua) → ③ 회사명 `주식회사 연:결 패밀리회사`. 자세한 HTML/CSS는 `assets/brand-footer.html` 참고.

## 5. 결과물 규칙 (항상)

- 언어: **항상 한국어**.
- 형식: **단일 HTML 파일** 하나 (외부 빌드 없이 더블클릭으로 열림). 여행·현장에서 폰으로 바로 쓰는 경우가 많으니 모바일에서 완벽히 동작할 것.
- **미리보기(preview) 항상 활성화** — HTML 포함 모든 아티팩트.
- 상태 저장은 JS 메모리 변수로(아티팩트 미리보기에서 `localStorage`가 막힐 수 있음). 영구 저장이 꼭 필요하면 사용자에게 별도 안내.
- 완성 후 `references/dopamine-patterns.md`의 마무리 체크리스트로 자가 점검.

## 빠른 빌드 순서

1. 유형 판별(0번) → 필요한 전용 스킬도 함께 읽기
2. 핵심 한 줄 정하기(1번)
3. 행동 메커니즘 + 도파민 7요소 중 4개+ 고르기(1·2번)
4. `references/dopamine-patterns.md`의 토큰·코드 패턴으로 빌드
5. `assets/brand-footer.html` 푸터 삽입(4번)
6. 마무리 체크리스트로 점검 → 단일 HTML, 미리보기 ON으로 전달
