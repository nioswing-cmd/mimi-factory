# 도파민 진단형 웹앱 빌더

선택형 진단(성격유형, 사랑의 언어, 심리테스트, 궁합 등) 웹앱의 표준 설계 규칙. 이 규칙을 어기면 "재미없거나" "신뢰할 수 없는" 진단이 된다.

## 절대 원칙 (이것이 전부다)

**숨김 + 실시간 시각화의 긴장 = 도파민**

1. **문제(선택지)에는 결과 단서를 숨긴다 — 조작 방지**
   - 선택지 라벨은 성향 글자(E/I, W/Q 등)나 결과를 암시하는 이모지를 절대 쓰지 않는다. 중립 라벨 `A` / `B`만 쓴다.
   - 선택지 배지·테두리 색을 결과별로 다르게 하지 않는다. 모두 동일한 중립색.
   - 선택지 위아래 순서를 매 문항 **무작위**로 섞는다. (위치로 학습하는 것까지 차단)
   - → 응답자는 "이걸 고르면 어떤 결과가 나오는지" 알 수 없어, 오직 문장 내용만 보고 솔직하게 답한다.

2. **그래프(게이지)로는 실시간 누적 결과를 보여준다 — 도파민**
   - 화면 상단에 누적 점수가 실시간으로 차오르는 게이지/슬라이더를 둔다.
   - 선택할 때마다 게이지가 즉시 움직이고, 현재 우세 유형을 미리 보여준다(예: 4축이면 `E · F P`처럼 완성돼 감).
   - → 결과가 "내 손으로 만들어지는" 감각이 도파민을 터뜨린다. 단, 선택지에 단서가 없으므로 보고도 조작은 불가능하다.

이 둘은 세트다. 게이지만 있고 선택지 단서도 있으면 → 조작 가능(나쁨). 선택지 단서도 없고 게이지도 없으면 → 밋밋함(나쁨). **반드시 "선택지 숨김 + 게이지 노출"을 동시에** 한다.

## 흔한 실수 (하지 마라)

- ❌ 진단 신뢰도를 높인다며 게이지까지 없애고 단순 진행률 바로 바꾸기 → 도파민이 죽는다. 게이지는 남긴다.
- ❌ 선택지에 결과 이모지/색/글자를 남겨두기 → 조작 경로가 된다.
- ❌ 결과를 문제 푸는 중간에 텍스트로 공개하기 → 게이지(시각적 암시)는 OK지만, 최종 유형 결과는 끝나야 공개한다.

## 추가 도파민 장치 (가능하면 다 넣어라)

- 선택할 때마다 누른 버튼 위치에서 작은 색종이 폭죽(canvas-confetti, 18개 정도)
- 결과 reveal 시 큰 폭죽 2연발
- 결과 막대그래프는 0→목표치로 슬라이드 인 애니메이션
- 궁합 % 는 0부터 카운트업
- 카드 등장 시 pop 애니메이션, 이모지 둥둥 떠다니기(float)
- `prefers-reduced-motion`은 존중해서 애니메이션 끈다

## 표준 화면 구조

1. **홈** — 후킹 카피 + "진단 시작" + "바로 궁합 매칭" 두 버튼
2. **진단** — [상단: 실시간 게이지 + 진행 카운트] [중단: 질문] [하단: A/B 중립 선택지(순서 셔플)]
3. **결과** — 유형 카드(이모지·색) + 점수 막대/슬라이더 + "당신은 이런 사람 / 이렇게 하세요 / 주의점" + 폭죽
4. **궁합** — 나×상대 선택(진단했으면 자동 채움) → %, 한 줄 진단, **닮은 점 / 조율 포인트 / 노력 체크리스트(다정하게)**. 진입·선택 즉시 갱신, 버튼 클릭 시 폭죽.

## 궁합 데이터 전략

- 조합 수가 적으면(예: 5×5=15) 조합별 텍스트를 직접 작성한다.
- 조합 수가 많으면(예: 16×16=256) 축별 일치/보완 **규칙으로 점수·조언을 생성**한다. 일일이 쓰지 않는다.
- 점수는 후하게, 조언은 항상 "안 맞는다"가 아니라 "이렇게 하면 서로를 채운다"는 다정한 코칭으로.

## 채점 무결성

- 선택지를 셔플해도 `pick(정확한_키)`로 채점하므로 결과는 정확하다. 표시(라벨·순서)와 채점(키)을 분리할 것.
- 강제선택 문항은 모든 축/짝이 **균등하게** 등장하도록 설계한다(예: 5개 짝 → 각 2회 = 20문항, 각 항목 만점 동일).

## 필수 마감 요소

- API 없이 100% 클라이언트에서 동작(외부는 폰트·confetti CDN만). `localStorage` 등 브라우저 저장소 사용 금지(메모리 변수/상태만).
- 단일 HTML 파일로 산출하고 미리보기 활성화.
- 하단에 브랜드 푸터 항상 삽입: "의미와재미공장" 미미팩토리 (주식회사 연:결 패밀리회사). 색·도트는 앱 테마에 맞춰 변주.

## 참고 구현

이 대화에서 만든 `love-language.html`(5색 누적 게이지)과 `mbti.html`(4축 슬라이더 + 실시간 4글자)이 이 규칙의 정석 구현이다. 새 진단 앱은 이 구조를 본떠 색·문항·결과 데이터만 교체한다.
