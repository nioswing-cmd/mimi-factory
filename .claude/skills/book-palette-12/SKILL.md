# 「12색 북 팔레트」 — 책의 정서를 색으로 입히는 시스템

책마다 같은 색이면 "매일 새 책" 컨셉의 수집 재미가 죽는다. 이 스킬은 **책의 장르·정서 → 12개 팔레트 중 1개**를 규칙적으로 매핑해, 시리즈 전체가 "다르지만 한 브랜드"로 보이게 만든다.

## 팔레트 데이터

원본 데이터는 `assets/palettes.json` (12개 팔레트 × 11개 CSS 변수). 빌드 시 이 파일을 읽어 적용한다. 12색 전체 미리보기는 `assets/palette-preview.html`.

## 선택 규칙 (우선순위 순)

1. **사용자가 팔레트를 지목**하면 그대로 쓴다. ("네온 느낌으로", "3번 팔레트")
2. **책의 핵심 정서**로 고른다 — 장르가 아니라 "읽고 난 뒤 남는 감정"을 기준으로. 예: 《미움받을 용기》는 심리서지만 정서는 '용기·추진'이므로 sunrise가 어울릴 수 있다.
3. 동률이면 **책 표지의 상징색**과 가장 가까운 팔레트.
4. 같은 저자/시리즈 연속 발행이면 **직전 책과 다른 팔레트**를 골라 수집 재미를 살린다.
5. 판단이 애매하면 사용자에게 후보 2개를 제시하고 고르게 한다.

## CSS 변수 규격 (11종 — 모든 신규 빌드 필수)

```css
:root{
  --bg:      /* 페이지 배경 (가장 어두움) */
  --bg2:     /* 그라디언트 보조 배경 */
  --card:    /* 카드·패널 배경 */
  --line:    /* 테두리·구분선 */
  --ink:     /* 본문 텍스트 (bg 대비 4.5:1 이상 보장됨) */
  --sub:     /* 보조 텍스트 */
  --accent:  /* 주 강조 (버튼·게이지·링크) */
  --accent2: /* 부 강조 (그라디언트 짝·호버) */
  --point:   /* 하이라이트 (점수·골드 포인트) */
  --ok:      /* 정답·성공 — 의미 고정(초록 계열) */
  --bad:     /* 오답·경고 — 의미 고정(붉은 계열) */
}
```

## 적용 원칙 (반드시 지킬 것)

- **색은 CSS 변수로만 쓴다.** 새 빌드에서 hex 하드코딩 금지 — 팔레트 교체가 `:root` 한 블록 교체로 끝나야 한다.
- **--ok / --bad의 의미는 절대 바꾸지 않는다.** 어떤 팔레트든 정답은 초록 계열, 오답은 붉은 계열. 도파민 피드백의 학습된 신호를 깨지 않기 위함이다.
- **다크 베이스 유지.** 12개 모두 어두운 배경 계열로 설계되어 미미팩토리 브랜드 톤과 몰입감을 지킨다.
- **대비 기준**: --ink는 --bg 대비 4.5:1 이상, --accent 위 텍스트는 어두운 색(#111 계열) 사용.
- **브랜드 푸터는 팔레트와 무관하게 유지** — `"의미와재미공장" 미미팩토리 (주식회사 연:결 패밀리회사)`.
- **기존 확정 앱(총·균·쇠 원조본)은 재도색하지 않는다.** 이 스킬은 신규 빌드부터 적용.

## quiz-to-read-a-book와 함께 쓰기

- 새 책 퀴즈 빌드 시: 레퍼런스 템플릿의 색 정의부를 위 11개 변수 기반 `:root` 블록으로 치환하고, 팔레트의 hex를 채운다. 홈 히어로의 그라디언트·게이지·마일스톤·폭죽 색도 --accent/--accent2/--point에서 파생시킨다.
- teaser/premium(v2.1) 잠금 배너·CTA는 --point 계열을 사용해 어떤 팔레트에서도 눈에 띄게 한다.
- 완료 리포트의 개념 카드 아이콘 배경은 --accent 15% 투명도, PDF 내보내기용 라이트 모드가 있는 경우 --ink/--bg를 반전하되 --ok/--bad는 유지.

## 확장 규칙

- 12개로 부족한 특수 정서(예: 공포·스릴러)가 반복 등장하면 13번째를 추가하지 말고, 기존 팔레트에서 --accent만 파생 변형한 "변주(variant)"로 처리해 시스템을 12개로 유지한다.
- 사용자 요청으로 새 팔레트를 만들 땐 palettes.json 규격(11개 변수)을 지키고 이 표에 행을 추가한다.
