# 퀴즈로읽는책 (quiz-to-read-a-book)

책 한 권을 **끝까지 익히게 만드는 학습 웹앱**을 만든다. 핵심 주장·맥락·키워드를 도파민 터지는 퀴즈로 반복하고, 마지막에 책 전체를 인포그래픽 리포트로 정리한다.

## 무엇을 만드는가 (산출물 정의)

단일 HTML 파일 1개. 4개 화면으로 구성:
1. **홈** — 후킹 카피 + 책의 상징 문장(인용) + "시작" 버튼
2. **퀴즈** — 상단 실시간 게이지 + 문항 + 선택지 + 푼 직후 해설
3. **결과** — 점수·정답률·등급 + "전체 요약 리포트 보기" 버튼
4. **리포트** — 책 전체 인포그래픽 요약(아래 7요소 필수)

## 절대 원칙 (이 두 가지가 핵심)

**① 도파민 = 실시간 시각화** + **② 반복 마스터 = 틀리면 다시**

- 상단에 **실시간 누적 게이지**를 둔다. 맞힐 때마다 즉시 차오르고, 진행도가 곧 책의 서사가 되도록 테마를 입힌다(예: 총균쇠는 '동서 축 원정' = 채집→농경→가축→문자→국가→완성).
- 정답 시 **색종이 폭죽**(canvas-confetti), 연속 정답 **콤보 보너스**.
- **틀린 문항은 큐 맨 뒤로 재투입** → 모든 문항을 한 번 이상 맞혀야 종료. "모두 익힐 때까지 반복"이 핵심.
- **재도전 유도**: 첫 시도 정답률로 등급 부여 + 세션 최고 점수 기록·경신 알림.

## 문항 설계 규칙

- 기본 **30문항**(사용자가 수를 지정하면 따른다). 난이도 1(쉬움)→2(보통)→3(어려움) **오름차순 정렬**해서 출제. 쉬운 기초→메커니즘·사례→심화·비판·요약 순.
- 각 문항: `{id, level, tag, q, options:[{t, ok}...], ctx}`.
  - 선택지는 4개, **정답은 정확히 1개**(`ok:true`). 표시할 때 **순서를 매번 셔플**하고 채점은 `ok` 플래그로 한다(위치 학습 차단).
  - **보기 길이 균형(필수):** 정답 보기가 늘 ‘가장 긴 보기’면 내용을 몰라도 길이만 보고 답을 찍을 수 있다. 4개 보기의 길이를 비슷하게 맞추고, **‘정답이 최장 보기인 문항’ 비율을 전체의 약 25~35%(무작위 수준)로 유지**한다. 오답은 짧은 단답("사랑이 식어서")이 아니라 정답과 비슷한 길이의 **그럴듯한 완성형 문장**("두 사람의 사랑이 진작에 식어 버려서")으로 쓴다. 작성 후 반드시 ‘정답=최장보기’ 비율을 측정해 검증한다.
  - `ctx` = **초보도 이해하는 쉬운 맥락 해설**. 푼 직후 자동 표시. 핵심어는 `<b>` 강조.
- 별도 `EXTRA[id] = {src, note}`:
  - `note` = 틀렸을 때 보여줄 **부연설명**(헷갈림 포인트/심화 1~2문장).
  - `src` = 그 문항과 관련된 **출처 키**(SOURCES 참조).

## 리포트 필수 7요소 (모두 포함)

1. **인과의 사슬 인포그래픽** — 책의 논리를 1줄 흐름도로(예: 궁극 원인 → 1단계 → 2단계 → 근접 원인 → 결과). 단계별 색상 카드 + 화살표.
2. **핵심 개념 카드** — 6~8개. 각 카드에 아이콘·1줄 설명·🔗 출처 링크.
3. **클릭해서 펼치는 목차 + 목차별 요약** — 아코디언. 헤더 탭 → 요약 패널 오픈(높이는 `scrollHeight`로 동적 계산). 책의 실제 부/장 구조를 반영. **요약은 한 덩어리 문단이 아니라, 이모지 + 짧은 한 줄(beat) 5~7개로 끊어서** 작성한다(숏폼에 익숙한 모바일 사용자용 스캔 가독성). 각 beat는 `{e:이모지, x:한 줄 설명}`, 한 항목 전체 분량은 300~1000자.
4. **책 속, 생각해볼 사례 3** — 카드 3개. 각 사례에 "왜 생각해볼 만한가" 한 줄.
5. **이 책이 우리에게 던지는 질문 3** — 카드 3개. 질문 + 성찰 힌트.
6. **틀린 부분 집중 복습** — 첫 시도 오답만 모아서 정답 + 💡부연설명(EXTRA.note) + 맥락(ctx) + 🔗출처. 모두 맞히면 "약점 없음" 축하 카드.
7. **섹션별 출처 링크** — 위 모든 항목에 관련 출처를 단다.
8. **리포트 PDF 저장(파일 다운로드, 무CDN)** — 리포트 화면에 ‘📄 PDF로 저장’ 버튼(`id="pdfBtn"`). claude.ai 미리보기 iframe은 외부 CDN/다운로드를 차단해 ‘콘텐츠 차단’ 페이지가 뜰 수 있으므로, **html2pdf.js 번들(약 0.9MB)을 HTML 안에 인라인 `<script>`로 내장**한다(외부 호출 0). 내장 시 번들 안의 `</script>`는 `<\/script>`로 이스케이프. 동작: 캡처 직전 `body.pdf-export #report`에 **밝은 테마용 CSS 변수**를 덮어쓰고 목차 아코디언을 모두 펼친 뒤 `html2pdf().set(opt).from(el).outputPdf('blob')`로 **Blob을 만들어 `<a download>` 클릭으로 직접 저장**한다(자동 저장이 막히면 `window.open(blobUrl)`로 새 탭, 그래도 실패 시 안내). `window.print()`는 iframe에서 막히므로 사용하지 않는다. (가장 확실한 사용처는 **다운로드한 HTML 파일을 실제 브라우저에서 여는 것**.)

추가로 심화 학습 링크(다큐·저자 강의 등)를 하단에 둔다.

## 출처 규칙 (중요)

- **반드시 web_search로 실제 존재가 확인된 URL만** 사용한다. 절대 URL을 지어내지 않는다.
- 우선순위: 한국어 위키백과 → 영문 위키백과(개념별 항목) → 신뢰도 높은 강의/다큐/학습 자료(LitCharts, EBS 등) → 나무위키(비판·논쟁 정리에 유용).
- `SOURCES = { key: {label, url} }` 형태로 모아두고, 카드·복습 항목에서 `srcLink(key)`로 단다.

## 도파민/채점 메커니즘 (구현 지침)

- 점수: `BASE_PTS = {1:10, 2:20, 3:30}`. 첫 시도 정답 = 만점 + 콤보 보너스, 복습(재시도) 정답 = 절반 점수.
- 콤보: 연속 정답마다 보너스 누적, 틀리면 0으로 리셋. `bestStreak` 기록.
- 마스터 수 = 한 번 이상 맞힌 문항 수. 게이지·마일스톤은 이 값으로 채운다.
- `firstWrongIds`(첫 시도 오답 Set)로 리포트의 "집중 복습"을 채운다.
- 등급 예: 첫 시도 정답률 90%+ / 75%+ / 55%+ / 그 이하 4단계, 격려형 문구.

## 수익화 빌드 모드 (구독/유료 배포용) — v2.1

「퀴즈로읽는책」을 **구독 상품**으로 배포할 수 있게, 모든 빌드에 엔진 JS 최상단 `APP_CONFIG` 블록을 둔다. 사용자가 모드를 지정하지 않으면 `mode:"free"`(기존과 동일 동작)로 빌드한다.

```js
const APP_CONFIG = {
  mode: "free",          // "free" | "teaser" | "premium"
  freeLimit: 10,          // teaser: 무료로 풀 수 있는 문항 수
  subscribeUrl: "https://blog.naver.com/4u_doumi",  // 구독 랜딩 URL (teaser/premium 필수) — 독서위키 블로그 (기본값, 변경 지시 없으면 이 값 사용)
  catalogUrl: "https://mimifactory.vibekr.com/quiz.html",  // '다른 책 퀴즈 보기' — 밤의 서재(독서퀴즈관) 메인 (기본값, 변경 지시 없으면 이 값 사용)
  utm: "utm_source=quizbook&utm_medium=app",
  edition: "",            // 예: "2026-07-02 · vol.128 데미안"
  accessCodeHash: ""      // premium 간이 게이트용 SHA-256 해시 (선택)
};
```

### 모드별 동작

- **free** — 전면 무료. 블로그 바이럴·샘플·무료 체험용. 기존 산출물과 동일.
- **teaser** — `freeLimit` 문항까지 정상 플레이 → 이후 **잠금 오버레이**(반투명 블러 + "나머지 N문항 + 완독 리포트는 구독자 전용" + `subscribeUrl`+utm CTA 버튼). 리포트는 7요소 중 **인과의 사슬 + 개념 카드 3장만 공개**, 나머지는 블러 처리 위에 동일 CTA. 게이지·점수는 잠금 후에도 보여서 "이만큼 남았다"를 시각화한다(도파민 결핍 = 전환 트리거).
- **premium** — 전체 공개. `accessCodeHash`가 있으면 시작 전 코드 입력 게이트(입력값을 SHA-256 해시해 비교, `crypto.subtle` 사용, 외부 의존 0).

### 산출물 파일 이름 규칙 (v2.3 확정 — 영문 통일)

파일명은 **영문(로마자) 소문자로 통일**한다. 한글 파일명은 사용하지 않는다(URL·서버·CDN 배포 호환).

- 형식: `title_author_quiz.html`(유료 premium), `title_author_quiz_teaser.html`(무료 free/teaser).
- **title / author = 로마자 소문자.** 제목 내부의 공백은 하이픈(`-`)으로 잇고, `title`·`author`·`_teaser` 사이 구분은 언더스코어(`_`)로 한다.
- 관사(`the`·`a`·`an`)는 생략 가능. 작가는 성(last name)만 쓰는 것을 기본으로 하되, 동명이인이 우려되면 `first-last`로 쓴다.
- 특수문자(쉼표·중점·콜론·따옴표 등)는 모두 제거한다. 한글 원제만 있는 국내서는 **널리 쓰이는 로마자 표기 또는 표준 로마자 표기**로 옮긴다.
- 2종 동시 빌드(premium + teaser) 시 동일 규칙: premium은 접미사 없음, 무료 노출본만 `_teaser`.

예시:
- 『The Untethered Soul』 / Michael A. Singer → `untethered-soul_singer_quiz.html`, `untethered-soul_singer_quiz_teaser.html`
- 『총, 균, 쇠』 / 재레드 다이아몬드 → `guns-germs-steel_diamond_quiz.html`, `guns-germs-steel_diamond_quiz_teaser.html`
- 『사피엔스』 / 유발 하라리 → `sapiens_harari_quiz.html`, `sapiens_harari_quiz_teaser.html`

### 유료 배포의 한계 고지 (반드시 이해하고 쓸 것)

- 단일 HTML의 잠금은 **클라이언트 사이드 심리적 장벽**이지 DRM이 아니다. 소스 보기로 우회 가능하다. **진짜 보호는 배포 채널에서** 한다: 구독자 전용 링크(스레드·메일·네이버 프리미엄콘텐츠·카카오 채널)로만 premium 빌드를 전달하고, 공개 채널에는 teaser 빌드만 노출한다.
- teaser 빌드에는 잠금 이후 문항의 정답·해설 데이터를 **아예 포함하지 않는 것을 권장**한다(데이터 자체를 freeLimit까지만 싣고, 나머지는 제목만). 이러면 소스 보기로도 유출이 안 된다. 이 경우 premium 빌드와 teaser 빌드는 **동일 reference에서 데이터 분량만 다른 2회 빌드**로 만든다.

### 전환·바이럴 장치 (teaser/premium 공통)

- **결과 공유 카드**: 결과 화면에 '🖼 결과 카드 저장' 버튼. canvas API로 자체 렌더(외부 CDN 0) — 책 표지풍 타이포 + 점수/등급 + 브랜드 푸터 + `subscribeUrl` 짧은 표기. `toBlob` → `<a download>` 저장.
- **시리즈 내비**: 푸터 바로 위에 `📚 다른 책 퀴즈 보기` 링크(`catalogUrl`+utm, 기본 목적지는 밤의 서재 quiz.html). 매일 1권 컨셉의 회유 동선.
- **에디션 표기**: 홈 화면 상단에 `edition` 문구(발행일·호수) — "매일 새 책" 컨셉의 수집 욕구 자극.

### 저작권 안전 규칙 (유료화 시 필수)

- 원문 **직접 인용은 홈 화면 상징 문장 1개, 짧게(한 문장 이내)**로 제한. 문항·해설·리포트는 원문 문장 복제가 아니라 **아이디어·사실·구조의 재구성**으로만 쓴다.
- 앱 하단에 고지 문구 고정: `본 앱은 도서의 핵심 개념을 학습 목적으로 재구성한 2차적 학습 콘텐츠입니다. 온전한 독서 경험은 원서로 만나보세요.` + 도서 구매 링크(제휴 링크 사용 시 "이 포스팅은 제휴 활동의 일환으로 수수료를 제공받을 수 있습니다" 고지 병기).
- 표지 이미지·본문 스캔은 절대 삽입하지 않는다(타이포·아이콘·색으로만 책 분위기 표현).

## 확정 규칙 (재발 버그 방지, v2.0)

- **MILESTONES는 반드시 배열**이어야 한다(객체로 바꾸면 게이지 렌더가 깨진다).
- **canvas-confetti CDN은 claude.ai 미리보기에서 차단**된다 → 폭죽은 **자체 canvas 구현**을 인라인으로 쓴다(외부 confetti CDN 금지).
- **html2pdf 인라인 번들은 다운로드 배포본에만** 포함한다. claude.ai **미리보기용 빌드에서는 번들을 제거**하고 PDF 버튼은 "다운로드한 파일에서 사용 가능" 안내로 대체한다(미리보기 용량·차단 이슈 회피).
- **`</body>`·`<body>` 앵커 삽입은 반드시 `rfind`(마지막 위치)로** 한다. 인라인 html2pdf 번들 안에 `</body>` **문자열 리터럴**이 있어, `replace(..., 1)`이나 첫 매칭 삽입을 쓰면 스크립트가 번들 한가운데 박혀 번들이 조기 종료·오염된다(v2.1 빌드에서 실제 발생·수정).

## 참고 구현 (정석)

`assets/reference-template.html` 은 이 스킬의 **정석 구현(『총, 균, 쇠』 30문항 버전)**이다. 새 책 앱은 이 파일을 복제한 뒤, 아래만 교체한다:

> **⚠️ reference-template.html 이 작업 환경에 없을 때 (중요):**
> 3파일 분리 리팩토링 등으로 컨테이너에 원본이 없을 수 있다. 이때 **추측으로 새로 만들지 말 것** — 디자인이 정석과 어긋난다. 다음 순서로 대응한다:
> 1. `conversation_search`/`recent_chats`로 가장 최근에 완성한 **다른 책의 산출물**(총·균·쇠 또는 그 파생본)을 찾는다.
> 2. 그 파일이 컨테이너에 없으면 **사용자에게 총·균·쇠(또는 직전 완성본) HTML 업로드를 요청**한다. 업로드본이 곧 이번 빌드의 reference가 된다.
> 3. 업로드본을 reference로 삼아 **데이터·고유 텍스트만** 교체한다(디자인/엔진/푸터/번들 불변). 직전 책의 고유명사는 `--extra-markers`로 반드시 함께 검사한다.

- `QUESTIONS` 배열(문항·해설), `EXTRA`(부연설명·출처키), `SOURCES`(검증된 URL)
- `MILESTONES`(진행 게이지 테마), 홈 화면 카피·인용
- 리포트의 `CONCEPTS` / `TOC` / `CASES` / `ASKS` 데이터
- 색 토큰(`:root`)으로 책 분위기에 맞는 팔레트

엔진(셔플·재투입·게이지·폭죽·리포트 렌더·아코디언)은 그대로 재사용한다. 구조를 바꾸지 말고 **데이터만 갈아끼우는 것**이 원칙이다.

`assets/verify_quiz.py` 는 산출물 **최종 검증 게이트**다. 빌드 후 present 직전 반드시 실행해 종료 코드 0을 확인한다(자세한 항목은 아래 ‘안전 빌드 절차 ⑤’).

## 진행 순서 (체크리스트)

1. 책 확정 → 핵심 주장·구조 파악(필요시 web_search).
2. 출처 URL을 web_search로 확보(SOURCES).
3. 30문항(난이도순) + ctx 해설 + EXTRA 부연설명 작성.
4. 리포트 7요소 데이터 작성(인과·개념·목차·사례·질문 + 출처).
5. **fresh `shutil.copy()`** 로 `reference-template.html` 복제 → 데이터 **1회만** 교체(아래 ‘안전 빌드 절차’ 준수).
6. JS 문법 검사(`node --check`) + 정답 1개 무결성 + **‘정답=최장보기’ 비율 25~35% 검증** + **구조 무결성(중복) 검사** + **잔존 참조키 검사** + 브랜드 푸터 정식 표기 확인.
7. 단일 HTML로 출력, 미리보기 활성화, 사용자에게 present.

## 흔한 함정과 안전 빌드 절차 (반드시 지킬 것)

매 작업마다 반복적으로 같은 버그가 났다. 아래는 그 원인과 회피법이다.

### ① 곡선따옴표(smart quotes) 때문에 str.replace가 NOT FOUND
템플릿 HTML 본문은 `“ ” ‘ ’`(곡선/스마트 따옴표)를 쓴다. 코드에서 곧은 따옴표 `" "`로 패턴을 만들면, 화면상 똑같아 보여도 **매칭이 안 돼 NOT FOUND**가 난다.
- 교체 대상 문자열은 **템플릿에서 그대로 복사**해서 곡선따옴표를 보존한다.
- 교체 직후 `assert old in html` 으로 매칭 성공을 확인하고, 실패 시 sed/정규식으로 우회한다.

### ② 삭제된 SOURCES 키 참조 → silent 런타임 오류
엔진 JS에는 **하드코딩된 참조키**가 있다. 대표적으로 deepen(심화 학습) 박스가 `const ebs = SOURCES.ebs; a.href = ebs.url`를 호출한다. SOURCES를 새 책으로 통째로 갈면서 **`ebs` 키를 빠뜨리면** 리포트 렌더 중 `Cannot read properties of undefined`로 조용히 깨진다(화면엔 티가 잘 안 남).
- **규칙: 새 SOURCES에도 `ebs` 키를 반드시 남긴다**(그 책의 심화 자료 URL로). deepen 박스 문구도 책에 맞게 바꾼다.
- 빌드 후 `grep -n "SOURCES\.\w\+" 엔진` 으로 **참조되는 모든 키가 SOURCES에 실제 존재**하는지 대조한다.

### ③ 데이터 블록 교체는 bracket-depth 파싱으로
`const NAME = [...]` 블록을 `\n];` 문자열 검색으로 끊으면, 중첩 구조 안의 `];`에 **조기 매칭**돼 블록이 잘린다.
- `const NAME = ` 시작점부터 여는 괄호(`[`/`{`)를 세고 **괄호 깊이가 0이 되는 지점**의 닫는 괄호 + 세미콜론까지를 한 블록으로 잡아 교체한다(문자열 리터럴 내부의 괄호·따옴표는 건너뛴다).

### ④ 반드시 fresh copy에서 1회만 교체 (문서 중복 방지)
이미 부분 수정된 파일에 빌드 스크립트를 다시 돌리면 구조 요소가 **누적·재삽입**되어 화면이 두 개로 겹쳐 보이는 중복 버그가 난다.
- 매 빌드는 **`shutil.copy('reference-template.html', work)`** 로 새로 시작하고, 모든 교체를 **딱 한 번** 수행한다.
- 구조 무결성 검사: 아래 다섯이 **모두 정확히 1**이어야 한다. 2 이상이면 파일이 오염된 것이므로 **처음부터 다시 빌드**한다.
  `<!DOCTYPE html>` · `<title>` · `</head>` · `<div class="wrap">` · `<section id="home"`
  > **⚠️ 번들 오탐 주의:** 인라인된 html2pdf 번들 안에는 `<body>`, `<!DOCTYPE`, `</html>` 같은 **문자열 리터럴**이 들어 있어, 단순 `count()`로 세면 2~5개로 나온다(정상). 진짜 DOM 중복 여부는 **번들이 없는 마커**로 판정하라 — 가장 확실한 건 `class="hero"`(홈 화면), `<section id="home"`, `<div class="wrap">`가 **각각 1개**인지다. `<body>`/`<!DOCTYPE`/`</html>` 개수만 보고 중복이라 단정하지 말 것.

### ⑤ 최종 게이트: `assets/verify_quiz.py`로 0 확인
present 직전, **`assets/verify_quiz.py`** 를 마지막 관문으로 돌린다(수동 점검이 놓치는 것까지 잡는다):

```
python3 assets/verify_quiz.py <완성본.html> [--extra-markers "직전책_고유명사" ...]
```

검사: ① 구조 무결성(중복) ② 문항 30·레벨 분포·난이도 오름차순·정답 1개 ③ ‘정답=최장보기’ 25~35% ④ SOURCES 참조키 존재 ⑤ 레퍼런스 잔존 텍스트 0(주석·랭크 문구 포함) ⑥ 브랜드 푸터. **종료 코드 0**을 확인한 뒤에만 present 한다. (작은/큰따옴표·공백 유무 모두 파싱. 직전에 만든 다른 책의 고유명사가 섞일까 걱정되면 `--extra-markers`로 함께 검사.)

> 이 스크립트는 실제로 싯다르타 산출물에서 미처 안 바꾼 랭크 문구 4개(`문명의 설계자`·`대항해 탐험가`·`농경 개척자`·`수렵채집인`)와 주석 `동서 축 마일스톤`이 남은 것을 잡아냈다. 눈으로는 놓치기 쉬우니 반드시 자동 검증한다.
